## Login & Profile Flow

For this sprint, I implemented the authentication-related functionality and protected routing flow.

I created:

* LoginPage
* RegisterPage
* ProfilePage

I also implemented:

* functional route guards (`canActivate`)
* shared `AuthService`
* signal-based authentication state
* navigation updates after login/logout
* localStorage authentication persistence

The Profile page is protected through `AuthGuard`, and unauthenticated users are redirected to the Login page using `UrlTree`.

Authentication state is managed reactively using Angular Signals.

## Problems

At first, I had difficulty understanding:

* how functional guards work together with signals
* how to organize shared authentication state
* how to structure routing integration between multiple feature areas
* how Angular Signals differ from regular component state


Additionally, ESLint configuration introduced several Angular-specific linting issues related to unused imports, outputs, and event naming.

---

## Solutions

I separated authentication logic into a shared `AuthService` and used Angular's `inject()` API inside functional guards.

For routing organization, I separated feature routes into dedicated routing files and connected them through lazy loading.

To improve authentication flow, I synchronized login state with `localStorage` using Angular `effect()`.


For ESLint issues, I cleaned up unused imports, renamed conflicting outputs, and updated components according to Angular ESLint rules.
