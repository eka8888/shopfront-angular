# Sprint 1 — Project Setup & Component Basics

## What I worked on

During Sprint 1, I worked on the initial Angular project setup and application architecture for our ShopFront project.

I created the base folder structure using a feature-based architecture approach with:

- core
- shared
- features
- layout

I also configured the routing foundation using standalone components and a lazy-loading-ready route structure.

My main implemented components were:

- HeaderComponent
- SearchBarComponent
- ProductCardComponent

### HeaderComponent

In HeaderComponent, I integrated the SearchBarComponent and implemented parent-child communication using output() event emission.

I also added navigation structure and connected the component to the main layout.

### ProductCardComponent

In ProductCardComponent, I implemented:

- input()
- output()
- event binding
- property binding
- interpolation
- ngOnInit()
- ngOnDestroy()

I used temporary mock product data passed from HomePage to test component communication and event handling.

## Problems

At first, I was confused about:

- how to separate layout components from feature components
- whether ProductCardComponent should receive data from services or temporary mock data
- how to organize scalable Angular architecture for team development

I also experienced Git merge conflicts while synchronizing the repository with GitHub.

## Solutions

I decided to create a separate MainLayout component and place the router-outlet there instead of directly inside AppComponent.

For ProductCardComponent testing, I used temporary mock product data passed through input() from the HomePage component.

To resolve Git conflicts, I cleaned up the README file and synchronized the repository history correctly using Git pull and merge workflows.

## What I learned

During this sprint, I learned:

- how standalone Angular architecture works
- how to structure a scalable Angular application
- how input() and output() work in Angular
- how parent-child component communication works
- how Angular lifecycle hooks work
- how routing and layouts are organized
- how to work with feature branches and pull requests in GitHub
